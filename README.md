📌 Why This Project
This project was created to make courier services more efficient by helping users find delivery agents at the lowest possible cost through a bidding system.
It also aims to simplify the work of courier agents by organizing requests and providing optimized routes using graph-based pathfinding, making deliveries faster,
easier, and more cost-effective.

🛠️ Tech Stack
The system is built using C/C++ with strong use of Data Structures such as Queue, Binary Search Tree, Min Heap, Graphs, and Hashing. File handling is used for persistent 
storage of users, agents, and requests. The entire system runs on a console-based interface without any frontend integration.

⚙️ How It Works
Users create courier requests which are stored in files and processed using a queue (FIFO). Agents register and place bids for deliveries, and the system selects the lowest
bid using heap/BST logic. Graph algorithms help in optimizing routes, making the system efficient, automated, and suitable for real-world courier operations.
